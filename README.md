## My Problem-Solving Approach

When solving a SQL problem, I generally follow this process:

1. Read the problem carefully and understand the expected output.

2. Identify the required tables, columns, keys, and relationships.

3. Understand the conditions, constraints, and possible edge cases.

4. Determine whether the problem requires filtering, joining, grouping, aggregation, subqueries, CTEs, or window functions.

5. Write an initial query based on the problem requirements.

6. Test the query against the provided examples and possible edge cases.

7. Check whether duplicates, `NULL` values, or grouping behavior can affect the result.

8. Review the query for correctness and readability.

9. Explore alternative approaches when they provide additional learning value.

10. Revisit the underlying concept so that I can apply it to similar problems in the future.

This process helps me focus on understanding SQL logic and developing reusable problem-solving skills instead of simply memorizing queries.
